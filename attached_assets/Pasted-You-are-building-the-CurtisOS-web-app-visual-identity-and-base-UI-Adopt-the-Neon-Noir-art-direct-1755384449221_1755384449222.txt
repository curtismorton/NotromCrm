You are building the CurtisOS web app visual identity and base UI. 
Adopt the “Neon Noir” art direction. Dark first. Calm by default. Punchy accents only where action is expected.

OUTPUT
1) A production-ready stylesheet at src/styles/curtisos.css that defines tokens, utilities, and component skins.
2) A small design-tokens module at src/styles/tokens.ts exporting the same values as a typed object.
3) Primitive React components in src/components that consume only the tokens and CSS classes. No UI libraries.
4) An AppShell layout and three example pages that demonstrate the identity: Dashboard, Inbox, Tasks. 
5) A README.md that documents how to use tokens, states, and motion.

TECH BASELINES
• React + TypeScript + Vite. 
• CSS Modules are fine. Global tokens must live in :root. 
• Respect prefers-reduced-motion and color-scheme. 
• All text meets WCAG AA 4.5:1 minimum. 
• Zero external fonts. Use system stack with sane fallbacks.

VISUAL IDENTITY. “NEON NOIR”
Scene and lighting
• Background uses a deep charcoal with a very soft vertical gradient plus a subtle vignette. 
• A tiny grain layer reduces banding. Keep it below 6% opacity.
• Only active or actionable elements glow. Everything else is matte.

State ramps
• Text on surfaces: ink-100 on surface-1 or darker. ink-300 for meta. 
• Borders: border-1 as default. border-2 for raised cards.
• Focus ring: 0 0 0 3px rgba(0,255,245,0.35) placed outside the element.
• Active glow: cyan ring with a soft outer blur only for primary actions and selected nav.

Typography
• System stack: Inter, SF, Segoe UI, Roboto, Ubuntu, Arial, sans-serif.
• Base size 16px. Line height 1.35. 
• Headings: 20 / 24 / 28 with weight 600. 
• Labels weight 500. Meta 12px with letter-spacing 0.2px.
• Use tabular numerals for metrics where available.

Spacing and radius
• Spacing scale in px: 8, 12, 16, 24, 32, 48.
• Card radius 14. Inputs radius 10. Pills radius 999.
• Grid rhythm 16.

Elevation and depth
• Two levels only.
  Level 0: surface-1 or surface-2. Border-1. No shadow.
  Level 1: surface-1. Border-2. Shadow 0 8px 24px rgba(0,0,0,0.35). 
• Hover adds a light inner highlight (inset 0 1px 0 rgba(255,255,255,0.06)). 
• Replace heavy shadows with a gentle “sheen” gradient on hover.

Motion
• Duration range 140–180ms. Use standard easing: cubic-bezier(0.2, 0.8, 0.2, 1).
• TranslateY 1–2px on hover for raised cards.
• Selection animates a 2px cyan edge that slides in from the left.
• Respect prefers-reduced-motion. If reduced, remove transforms and glows.

Utility layers to implement in CSS
• .bg-scene adds the vertical gradient and vignette via pseudo elements.
• .noise applies a very subtle grain overlay.
• .glow-cyan applies the active ring: box-shadow 0 0 0 2px rgba(0,255,245,0.25), 0 0 24px rgba(0,255,245,0.18).
• .glass-pill builds a translucent pill: background rgba(255,255,255,0.08), border rgba(255,255,255,0.18), inner highlight rgba(255,255,255,0.06).
• .capsule-bar styles progress bars with rounded ends and a cyan fill segment.

Core skins
Buttons
• Primary: surface-2 background with cyan glow on focus and active. Label uses ink-100. Hover adds inner highlight and lifts by 1px. Disabled removes glow and uses ink-400.
• Secondary: surface-2 with border-1. Hover only inner highlight. 
• Destructive: danger-500 text with border rgba(255,77,109,0.4). No red fills.

Inputs
• Background surface-2. Border-1. On focus apply focus ring and slight outer glow. 
• Placeholder ink-400. 
• Support a compact density flag that reduces padding by 2px all around.

Chips and badges
• Use .glass-pill with text in ink-100 for selected, ink-300 for default.
• Priority chips color accents: Urgent danger-500 text. High warn-500. Medium ok-500. Low action-cyan-500.

Cards
• Use Level 1 elevation. Border-2. 
• Title at 14–15px ink-200. Meta at 12px ink-400.
• Optional left priority spine 2px wide for task cards.

Focus and keyboard
• Every interactive element must support :focus-visible with the cyan ring.
• Do not rely on color only for state. Add icons or spines where needed.

Iconography
• Duotone look. Idle is outline with ink-300. Active adds a soft inner glow using action cyan. 
• Use only SVGs and currentColor for tinting.

Charts
• Lines 2px with rounded caps. Use action cyan for primary. 
• Areas use momentum pink at low alpha for subtle depth. 
• Axis labels ink-400. Gridlines rgba(255,255,255,0.05).
• Small “spark” charts inside metric cards combine a blurred pink area with short cyan bars.
